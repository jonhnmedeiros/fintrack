# Decision Log — FinTrack PRD

## 2026-05-25

### Decisão: Escolha do modo Capability-First
**Contexto:** Usuário optou por estrutura baseada em funcionalidades (Visão + Features) em vez de Personas + Jornadas.
**Decisão:** Seguir capability-first com features agrupadas e FRs aninhadas.

### Decisão: Papéis de usuário
**Contexto:** Necessidade de multiusuário com permissões distintas.
**Decisão:** Dois papéis — Titular (CRUD completo) e Visualizador (read-only). Convite por email.

### Decisão: Autenticação
**Contexto:** Manter ou substituir NextAuth.
**Decisão:** Manter NextAuth (Auth.js) com PrismaAdapter e Credentials Provider, JWT.

### Decisão: Cadastro de usuário
**Contexto:** Não existe fluxo de sign-up atualmente.
**Decisão:** Implementar cadastro com email e senha. Confirmação de email a definir.

### Decisão: API de cotações
**Contexto:** Necessário para rentabilidade e preço atual dos ativos.
**Decisão:** Adiado para decisão na arquitetura. Google Finance não tem API oficial; considerar Yahoo Finance, Alpha Vantage ou Brasil API.

### Decisão: Importação de notas de corretagem
**Contexto:** Portabilidade de preço médio entre corretoras.
**Decisão:** Formato PDF, suporte inicial para BTG Pactual e Inter. Exportação em CSV/JSON.

### Decisão: Escopo MVP
**Contexto:** O que entra vs. o que fica para depois.
**Decisão:** Metas e Configurações ficam para v2. Investimentos e importação entram no MVP.

### Decisão: API de cotações
**Contexto:** Necessário para rentabilidade e preço atual dos ativos.
**Decisão:** Utilizar API gratuita. Yahoo Finance, Brasil API ou similar. Decidir na arquitetura.

### Decisão: Cadastro sem confirmação de email
**Contexto:** Simplicidade para uso pessoal e familiar.
**Decisão:** Apenas email + senha, sem verificação de email.
